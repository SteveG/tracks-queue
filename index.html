tracks.cute

What is it?

Tracks.cute is a (partial) reimplementation of Tracks in Python and Qt.

Quoting getontracks.org:

Tracks is a web-based application to help you implement David Allen’s Getting Things Done™ methodology. It was built using Ruby on Rails, and comes with a built-in webserver (WEBrick), so that you can run it on your own computer if you like. It can be run on any platform on which Ruby can be installed, including Mac OS X, Windows XP and Linux. Tracks is Open Source, free and licensed under the GNU GPL.

How do I get it?

Current there are no packaged releases of tracks.cute. The current source code can be retrieved using the following command.

git clone git://github.com/SteveG/tracks.cute.git

There are several dependencies. These are:

  • Python 2.6 or 2.7
  • PyQT 4.6 or 4.7

It is also current assumed that you will have an existing tracks database, in sqlite format.

How do I use it?

WARNING: tracks.cute is pre-pre-alpha software, use at your own risk

Navigate to the src directory and execute tracks.py as follows:

./tracks.py

Tracks.cute will then ask you to select your database file. Whilst it will claim that it can create one for you this hasn't been implemented yet and will fall over.

Finally tracks.cute will ask you to nominate a user from within the database file.

tracks.cute

what is it?

Tracks.cute is a (partial) reimplementation of Tracks in Python and Qt.

Quoting getontracks.org:

Tracks is a web-based application to help you implement David Allen’s Getting Things Done™ methodology.

Screenshot of context listThe main page shows all your actions sorted by context, but you can also view pages showing just the actions in one context (for when you want to focus on your office tasks, email or errands), or just the actions in one project.

A form on the main page lets you add actions to any context or project, whereas forms on context and project pages automatically assign the current context or project to the new next action.

Actions can have due dates. The ‘tickler’ allows you to schedule actions to be shown on the home page after some future date. This allows you to get the action out of your head into a concrete, safe system (where it belongs), but you don’t need to be bothered by the action until the time comes when you can actually deal with it.

how do I get it?

There are no packaged releases of tracks.cute. The current source code can be retrieved using the following command.

git clone git://github.com/SteveG/tracks.cute.git

Tracks.cute uses:

  • Python 2.7
  • PyQt 4.7

tracks.cute utilises the SQLite database from tracks. It should be able to open an existing tracks SQLite database file.

how do I use it?

WARNING: use of tracks.cute is at your own risk

Navigate to the src directory and execute tracks.py as follows:

./tracks.py

Tracks.cute will then ask you to select your database file. If you don't have an existing SQLite database file tracks.cute will create a new one using a template.

Finally tracks.cute will ask you to nominate a user from within the database file.
